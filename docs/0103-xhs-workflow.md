# 小红书内容创作全流程设计

> 基于 `planning-with-files` 模式，使用持久化文件管理工作记忆，实现从主题定义到发布复盘的完整闭环。

**版本**：v3.8
**创建时间**：2026-03-15
**最后更新**：2026-03-18（第十二轮优化：图片生成改为离线操作——先给prompt和编号，手工上传图片后再发文）
**优化说明**：将AI生图改为离线操作流程，避免自动生成图片不可控的问题
**核心调整**：阶段9输出图片prompt+编号，阶段10需确认图片已上传后再发布

---

## ⚠️ 执行规范（重要）
0. ***工作目录**：/root/Obsidian/claw/小红书运营/项目标题
1. **文档编号**：每个生成的文档前面加上编号 01、02、03...，编号与阶段步骤一一对应
2. **工具检查**：如果调用工具失败，必须实时告知用户，先确保工具可用再往下执行
3. **实时汇报**：完成每一个子步骤都要汇报，不要一次性全部完成
4. **发布确认**：不要自动发布文章到平台，必须得到用户确认后才能执行发布
5. **阶段反馈**：每个阶段完成后，根据结果判断是否需要调整上一阶段的内容
6. **定稿确认**：必须生成最终可发布版本（09_review_final.md），用户确认后才能进入发布阶段
7. **图片离线操作流程（v3.8核心）**：
   - 阶段9：提供图片prompt+编号（如：封面图-01、配图-01等）
   - 用户：根据prompt生成图片，手工上传到指定文件名
   - 阶段10：确认图片已上传后再发布
8. **信息溯源**：所有引用数据必须标注来源URL，搜索结果需要保留原始链接
9. **用户确认**：每个阶段完成后需要用户确认结果，再进入下一阶段
10. **文件隔离**：每个项目独立目录，按主题名称创建
11. **账号数据源说明**：无法确定时需与用户沟通确认
12. **AI辅助合规（前置）**：AI辅助创作必须保留"真人感"，禁止全AI托管模式，适用于全流程所有阶段
13. **真实性要求**：内容必须基于真实体验和经历，禁止纯AI生成无真人参与
14. **真人审核要求**：所有AI生成内容必须经过真人审核，不得直接发布AI输出
15. **素材积累**：竞品调研后立即建立素材库，支撑后续内容矩阵化生产
16. **选题验证**：发布前验证选题是否具备爆款潜力
17. **采集数量下限**：阶段04信息采集总量不得少于100篇，各平台分配数量需达标
18. **正文采集要求**：必须采集完整正文内容，不得仅采集标题和摘要
19. **循环执行机制**：阶段04/05/06采用循环子步骤，避免单次执行超时
20. **图片确认机制（v3.8新增）**：阶段10发布前必须确认图片已上传到指定路径，否则不得发布

## 🔄 循环子步骤执行机制

### 4.1 为什么要循环执行 + 即时提取

**阶段4面临的双重挑战**：
- **Token超限**：大量正文内容超出单次对话容量
- **链接失效**：小红书URL平均有效期仅24-48小时，其他平台也可能失效

**解决方案**：采用「搜索→提取一体化」循环机制
- **边搜边提**：发现合适内容立即提取，绝不先存URL再处理
- **分批执行**：每批10-15篇，避免token超限
- **即时保存**：每批完成后立即保存，不留待处理队列

- 确认工具正常后继续，如工具不可用，记录问题并尝试替代方案

#### 子步骤B：关键词调用 ✅
- 调用阶段3核心关键词（3-5个） → **汇报进度**
- 根据初步搜索结果补充长尾关键词 → **汇报进度**
- 记录本轮使用的关键词组合

#### 子步骤C：分平台采集 ✅
